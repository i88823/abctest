httpd编译安装的方式

==================

第一步，下载源码包

# cd /usr/local/src/
下载源码包：
wget https://mirrors.tuna.tsinghua.edu.cn/apache//apr/apr-1.6.5.tar.gz
wget https://mirrors.tuna.tsinghua.edu.cn/apache//apr/apr-util-1.6.1.tar.gz
wget https://mirrors.tuna.tsinghua.edu.cn/apache/httpd/httpd-2.4.37.tar.gz
备注：三个源码包都放在/usr/local/src/

第二步，安装arp-1.6.5及arp-util-1.6.1

# cd /usr/local/src/
# cp -r apr-1.6.5  /usr/local/src/httpd-2.4.37/srclib/apr
# cd apr-1.6.5
#./configure --prefix=/usr/local/apr
# make && make install

# cp -r apr-util-1.6.1  /usr/local/src/httpd-2.4.37/srclib/apr-util
# cd /usr/local/src/apr-util-1.6.1
# ./configure --prefix=/usr/local/apr-util --with-apr=/usr/local/apr
# make && make install
遇到中间报以下错误：
libtool: warning: remember to run 'libtool --finish /usr/local/apr/lib'
libtool --finish /usr/local/apr/lib
使用以下命令：
# make clean
# make && make install   就OK了！
编译apr-util-1.6.0的时候遇到一个错误：
xml/apr_xml.c:35:19: fatal error: expat.h: No such file or directory
在网上查找说可能是少了expat库，
yum install expat-devel安装该库，安装完再次编译，果然可以
作者：程序猿吧你
来源：CSDN
原文：https://blog.csdn.net/hejinyin/article/details/77701702
版权声明：本文为博主原创文章，转载请附上博文链接！

第三步，安装httpd-2.4.37

# cd /usr/local/src/httpd-2.4.37
# ./configure --with-included-apr --prefix=/usr/local/apache2.4 --with-apr=/usr/local/apr --with-apr-util=/usr/local/apr-util --enable-so --enable-mods-shared=most
# make && make install

第四步，编写启动脚本

vim httpd

#!/bin/bash
# chkconfig: 12345 80 90
function start_http()
    {
        /usr/local/apache2.4/bin/apachectl  start
    }
function stop_http()
    {
         /usr/local/apache2.4/bin/apachectl  stop
    }
case "$1" in
        start)
        start_http
    ;;  
    stop)
    stop_http
    ;;  
    restart)
    stop_http
    start_http
    ;;
    *)
    echo "Usage : start | stop | restart"
    ;;
esac

第五步，给脚本设置权限及启动服务

加入系统服务：
# chmod  a+x  httpd
# cp  -arf  httpd  /etc/init.d/
启动自己编写的服务：
# systemctl  daemon-reload
# systemctl  start  httpd
设置开机自启动：
# chkconfig  --add  httpd
查看httpd 是否加入
# chkconfig --list httpd
配置文件路径：
/usr/local/apache2.4/conf/httpd.conf

# chkconfig --list httpd

注：该输出结果只显示 SysV 服务，并不包含
原生 systemd 服务。SysV 配置数据
可能被原生 systemd 配置覆盖。 

  要列出 systemd 服务，请执行 'systemctl list-unit-files'。
  查看在具体 target 启用的服务请执行
  'systemctl list-dependencies [target]'。

httpd              0:关    1:开    2:开    3:开    4:开    5:开    6:关
查看 httpd进程
# ps -aux | grep httpd
查看 80端口
# netstat -antlp | grep 80
  tcp6       0      0 :::80                   :::*                    LISTEN      1771/httpd
查看 httpd程序位置
# ls -l /proc/1771/exe
lrwxrwxrwx. 1 root root 0 12月 15 15:08 /proc/1771/exe -> /usr/sbin/httpd

# whereis httpd
httpd: /usr/sbin/httpd /usr/lib64/httpd /etc/httpd /usr/share/httpd /usr/share/man/man8/httpd.8.gz

Ubuntu LAMP下搭建wordpress

准备 LAMP 环境

=================

LAMP动态网站部署架构是一套由Linux + Apache + MySQL + PHP组成的动态网站系统解决方案，具有免费、高效、扩展性强且资源消耗低等优良特性。
1.安装 Apache2
apt-get install apache2 -y
2.安装 PHP 组件
apt-get install php7.0 -y
安装 php 相关组件:
apt-get install libapache2-mod-php7.0
3.安装 MySQL 服务
apt-get install mysql-client mysql-server
4.重启 MySQL 服务和 Apache 服务
sudo service mysql restart
5.wordpress网站文件放置
下载
wget https://wordpress.org/latest.tar.gz
解压
tar -xvf latest.tar.gz
移动源码到网站的根目录
mv wordpress /var/www/html/
6.更改根目录权限
chmod -R 777 /var/www/html/wordpress/
7.为 wordpress 配置一个数据库
进入 mysql，输入以下代码后，按提示输入您MySQL密码:
mysql -u root -p
输入密码
show databases; 查看数据库
create database wordpress;
show databases;
quit; 退出
浏览器访问wordpress配置
http://服务器ip/wordpress/
如果密码错误就改密码
修改mysql密码
service mysql stop
mkdir -p /var/run/mysqld
chown mysql:mysql /var/run/mysqld
/usr/bin/mysqld_safe –skip-grant-tables –skip-networking
开启新终端
masql -u root
use mayql;
update user set authentication_string=PASSWORD(“这里是要改的密码”) where user=’root’;
update user set plugin=”mysql_native_password”;
flush privileges;
exit
进入http://服务器ip/wordpress/ 修改配置
